Chapter 1

준비물 챙기기 🎒

시작하기 전에 딱 두 가지만 깔면 돼요

🪟 윈도우 사용자는 여길 먼저 봐주세요!

다음 슬라이드에서 WSL + Ubuntu 설치부터 진행해주세요.

파워셸(관리자)에서 아래 명령어 한 줄이면 끝!

파워셸 (관리자) wsl --install -d Ubuntu

이것만 깔면 그 다음은 맥과 동일해요. → 다음 슬라이드에서 상세 안내!

⚙️ 윈도우·맥 둘 다

Node.js자동차한테 엔진이 필요하듯, 오픈클로한테 Node.js가 필요해요. 한 번만 깔면 끝이고, 이후엔 신경 안 써도 돼요!

오픈클로의 엔진이에요.
이게 있어야 오픈클로가 돌아가요. 한 번만 깔면 끝!

API 키? 그런 거 필요 없어요! 🎉
오픈클로는 OAuth카카오 로그인, 구글 로그인 해보셨죠? 그거예요! 브라우저에서 "허용" 버튼 한 번 누르면 끝. 비밀번호 따로 만들 필요 없어요! 로그인을 지원해서, 나중에 브라우저에서 로그인만 하면 돼요.
맥 유저는 Node.js만 챙기면 끝!
윈도우 유저도 명령어 2줄이면 설치 끝나요 🐾

모르는 용어가 나오면 밑줄 친 단어에 마우스를 올려보세요.
쉬운 설명이 팝업으로 나올 거예요!
Chapter 1-1 · 윈도우만

🪟 윈도우 사용자는 여기부터!

윈도우에서 오픈클로 설치, 생각보다 훨씬 간단해요.
wsl --install -d Ubuntu 명령어 한 줄로 리눅스 환경이 세팅되고,
그 다음은 맥과 완전히 동일합니다!

🍎 맥 사용자는 이 페이지 스킵! → "다음" 버튼 누르세요

1 파워셸을 관리자 권한으로 실행

키보드에서 Windows 키 → "PowerShell" 검색 → "관리자로 실행" 클릭

2 WSL + Ubuntu 설치 (명령어 한 줄!)
파워셸 (관리자) wsl --install -d Ubuntu

WSL과 Ubuntu가 함께 설치돼요. 끝나면 컴퓨터를 재시작하세요.

3 재시작 후 Ubuntu 열기

시작 메뉴에서 "Ubuntu" 검색 → 클릭하면 터미널이 열려요.

처음 실행하면 사용자 이름/비밀번호를 만들라고 해요:

  • 사용자 이름: 영어 소문자 (예: myname)
  • 비밀번호: 아무거나 (입력할 때 화면에 안 보이는 게 정상!)
4 오픈클로 설치 (이것도 한 줄!)
Ubuntu 터미널 curl -fsSL https://openclaw.ai/install.sh | bash

Node.js가 없으면 자동으로 설치해줘요. 약 7분 정도 걸려요.

✅ 이게 끝이에요!
WSL 설치 + 오픈클로 설치 = 명령어 딱 2줄.
설치가 끝나면 자동으로 온보딩이 시작돼요 → 이 가이드의 Chapter 3 (온보딩)부터 이어서!
🔧 혹시 안 되나요? (클릭하면 열려요)
❌ "가상 머신 플랫폼" 오류 / WslRegisterDistribution failed

BIOS에서 가상화(VT-x / AMD-V)가 꺼져있을 때 나와요.

→ 컴퓨터 재부팅 → BIOS 진입(보통 F2/DEL) → Virtualization Technology를 Enabled로 변경 → 저장 후 재부팅

❌ wsl --install 자체가 안 먹어요 (명령어를 찾을 수 없음)

Windows 10 버전 2004 미만이면 WSL 명령어 자체가 없어요.

→ 설정 → 시스템 → 정보에서 버전 확인! → Windows Update로 최신 버전 업데이트 먼저!

→ 그래도 안 되면 코난쌤의 WSL 상세 가이드 또는 리도님의 윈도우 가이드를 따라해보세요.

❌ Microsoft Store 관련 오류 (0x80070422 등)

회사/학교 PC에서 Microsoft Store가 정책으로 막혀있을 때 발생해요.

→ 파워셸(관리자)에서 수동 설치:
wsl --install -d Ubuntu --web-download
Store를 거치지 않고 웹에서 직접 다운로드해요.

❌ Ubuntu가 시작 메뉴에 안 보여요

→ 컴퓨터를 재시작했는지 확인! 재시작 후에도 안 보이면 Microsoft Store에서 "Ubuntu"를 검색해서 직접 설치하세요.

❌ curl 명령어가 안 먹어요

→ 파워셸이 아니라 Ubuntu 터미널에서 실행해야 해요! 시작 메뉴에서 "Ubuntu"를 열어주세요.

위 방법으로 해결이 안 되면, 스크린샷과 함께 하나하나 따라할 수 있는 가이드가 있어요!

📘 코난쌤의 WSL 설치 상세 가이드 — 가상 머신 플랫폼 활성화, 커널 업데이트까지 단계별 설명
📘 Reedo(리도)님의 윈도우 전용 상세 가이드 — 보안 설정까지 윈도우 전용 5단계 가이드
진짜 명령어 2줄이에요! 맥이랑 다를 게 없어요 🐾
혹시 막히면 코난쌤, 리도님 가이드가 든든한 백업이 되어줄 거예요!
Chapter 1-2

Node.js 설치하기

오픈클로가 돌아가려면 Node.js 24(권장)가 필요해요.

✅ 별도 설치 필요 없어요!

앞 단계에서 curl -fsSL https://openclaw.ai/install.sh | bash 를 실행하면
Node.js가 없을 경우 자동으로 설치해줘요.

이미 Chapter 1-1을 완료했다면 → 바로 Chapter 3 (온보딩)으로!
1 터미널 열기
Cmd + Space → "터미널" 또는 "Terminal" 입력 → Enter
2 Homebrew맥에서 프로그램을 쉽게 깔아주는 도우미예요. 앱스토어처럼 "이거 깔아줘!" 하면 알아서 깔아줘요. 근데 마우스 대신 글자로!로 설치 (권장)

Homebrew가 없는 경우, 먼저 이걸 복사해서 터미널에 붙여넣기:

터미널 /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

비밀번호를 물어보면 맥 로그인 비밀번호를 입력하세요. (화면에 안 보이는 게 정상!)

Homebrew 설치 후 (또는 이미 있는 경우):

터미널 brew install node
3 설치 확인
터미널 node --version
v24.x.x ← 숫자가 나오면 성공! 🎉
🔧 여기서 막히는 경우 (클릭하면 열려요)
❌ "brew: command not found"

Homebrew 설치가 안 된 거예요.

→ 해결법:

위의 Homebrew 설치 명령어를 먼저 실행하세요. 설치 후 터미널을 껐다 다시 열어야 인식돼요!

Apple Silicon 맥(M1/M2/M3/M4)이면 추가로 이것도 실행:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
❌ "xcode-select: note: no developer tools were found"

Xcode 명령줄 도구가 필요해요.

→ 해결법: 팝업이 뜨면 "설치" 클릭. 안 뜨면 수동으로:

xcode-select --install
❌ "node: command not found" (설치했는데 안 됨)

→ 해결법: 터미널을 완전히 껐다가 다시 열어보세요.

그래도 안 되면 nodejs.org에서 .pkg 파일을 직접 다운받아 설치해도 돼요.

Homebrew 없이도 nodejs.org에서 직접 .pkg 파일을 다운받아 설치할 수 있어요!
